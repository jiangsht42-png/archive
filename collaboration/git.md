# Git 是什么

Git 是一个分布式版本控制系统，用于跟踪、管理和协作开发代码或其他文本内容。它以快照方式记录项目历史，允许多人并行工作并安全合并。

## 核心特性
- 分布式：每个仓库都是完整副本，可本地提交、查看历史、创建分支。
- 快照记录：提交保存的是文件快照，便于快速回滚、比较与恢复。
- 轻量分支：分支创建、切换和合并成本低，鼓励实验和并行开发。
- 安全性：对象哈希校验（SHA-1/SHA-256）确保历史记录完整与防篡改。
- 协作友好：通过远程仓库推送/拉取共享变更，支持多人协作。

## 常见工作流
1. 克隆（clone）远程仓库到本地。
2. 创建分支（branch）开展新特性或修复，避免直接改主分支。
3. 本地提交（commit）保存每个小步变更，并写清晰的提交说明。
4. 拉取（pull/pull --rebase）同步他人更新，减少冲突。
5. 合并或变基（merge/rebase）整合分支成果。
6. 推送（push）把本地提交发布到远程供团队使用。

## 适用场景
- 软件开发：管理源代码、配置与脚本。
- 文档协作：追踪方案、规范、文档的演进。
- 学习与实验：通过分支尝试方案，随时回到稳定版本。

## 为什么选择 Git
- 性能好：本地操作无需网络，提交、分支、比对都很快。
- 生态成熟：GitHub/GitLab 等平台提供代码托管、评审与 CI/CD。
- 社区广泛：开源、免费、标准化工具链，学习资料丰富。
